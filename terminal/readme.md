# MUF ターミナルユニット
このユニットは、MUF（Memory Unit Framework）のネットワークに参加し、共有メモリに対して人間が直接データを書き込んだりリクエストを発行したりするための操作用端末です。産業用システムの操作盤やHMI（Human Machine Interface）のような役割を担い、開発時の動作確認やデバッグにおいて能動的にシステムの状態を変化させるために使用します。

ユニットの起動は、プロジェクトのルートディレクトリにおいて

> docker compose up -d

を実行することで行います。これにより、ターミナルユニットはバックグラウンドで起動し、いつでも入力を受け付けられる待機状態になります。一度起動してしまえば、コンテナ自体を停止させるまで常駐し続けます。

実際に共有メモリを操作するための対話モードへ入るには、次のコマンドをターミナルで実行してください。

> docker exec -it muf-terminal python terminal.py

というコマンドを入力することで、コンテナ内で動作しているプログラムの入力インターフェースへと直接接続されます。接続に成功すると「MUF Terminal > 」というプロンプトが表示され、自由に入力が行えるようになります。

操作の基本形式は、書き込みたい対象のパスと、送信するデータをスペースで区切って一行で入力するスタイルです。例えば、テスト用の領域に文字列を書き込みたい場合は「muf/test/unit hello_world」のように入力し、エンターキーを押下します。これにより、SDKを通じて共有メモリのフラグ管理とデータ書き込みが実行されます。もし別のターミナルでモニターユニットを起動していれば、この操作の瞬間にメモリ上の値が更新される様子を視覚的に確認できます。

対話モードを終了して通常のシェルに戻りたい場合は、プロンプトに対して「exit」または「quit」と入力するか、キーボードのCtrl+Cを押下してください。対話モードを終了してもコンテナ自体はバックグラウンドで動き続けているため、再度コマンドを実行すればいつでも操作を再開することが可能です。
